## 1. 硬體準備

### 必備
- PEL-5000C 主機（規格依需求：PEL-5006C 6kW / PEL-5012C 12kW / PEL-5024C 24kW）
- 待測 DC 發電機（DUT）
- 適合的電流線（依 DUT 額定電流決定線徑，建議 30% 餘裕）
- LAN 網路線 (Cat 5e 以上) — 連到 PC 或 Switch
- 電腦（Windows / Linux 均可，本框架以 Windows 11 為主）

### 接線
```
DC發電機 (+) ──→ PEL-5000C 前面板 (+)
DC發電機 (–) ──→ PEL-5000C 前面板 (–)

PEL-5000C 後背板 LAN 埠 ──→ Switch / PC
```

> 高電流場合務必使用銅排或粗銅線，並鎖緊端子。鬆動的接點會造成壓降誤差與發熱。

## 2. 網路設定

### PEL-5000C 端
1. 開機後按 `Menu` → 進入 `System` → `LAN`
2. 確認以下參數：
   - IP Address: `192.168.0.100`（出廠預設）
   - Subnet Mask: `255.255.255.0`
   - Gateway: `192.168.0.1`
   - Socket Port: `4001`（不可改）

### PC 端
方法 A — 直連 PC（最簡單）
1. PC 網卡設靜態 IP：`192.168.0.50` / Subnet `255.255.255.0`
2. PEL-5000C 與 PC 用網路線直接相接
3. 開 PowerShell 測：`ping 192.168.0.100`

方法 B — 走公司網路
1. 把 PEL-5000C 的 IP 改成同網段的可用位址（找 IT 要）
2. 同步更新 `config.yaml` 中的 `instrument.ip`

## 4. config.yaml 設定說明

### `instrument`（連線參數）
| 欄位 | 說明 | 預設 |
|---|---|---|
| `ip` | PEL-5000C 的 IP | 192.168.0.100 |
| `port` | TCP socket 埠 | 4001 |
| `timeout` | 通訊逾時 (秒) | 5.0 |
| `command_delay` | 每筆指令後等待 (秒) | 0.05 |
| `read_buffer` | 接收緩衝大小 | 4096 |

### `dut`（待測物規格）
| 欄位 | 說明 |
|---|---|
| `name` | DUT 識別名（會寫入報告） |
| `rated_voltage` / `rated_current` / `rated_power` | 額定值 |
| `voltage_max` / `current_max` | **安全保護上限** — 超過會自動 LOAD OFF |

### `vi_curve`（V-I 掃描）
| 欄位 | 說明 |
|---|---|
| `current_start / stop / step` | CC 掃描起點 / 終點 / 步階 (A) |
| `dwell_seconds` | 每點變更電流後等待穩定的秒數 |
| `samples_per_point` | 每點重複量測次數（取平均） |
| `sample_interval` | 重複量測之間的間隔 |

### `efficiency_test`（效率測試）
- `current_points`：要量測的電流點清單（不規則步階用）
- `input_power_source`：
  - `manual` — 每點程式會問你 `P_in` 是多少
  - `constant` — 全程使用 `input_power_constant`
  - `external_meter` — 預留接外部儀器（需自行擴充）

### `load_sweep`（多模式掃描）
- `modes`：要跑的模式列表，可選 `["CC","CR","CV","CP"]`
- 各模式的 `[start, stop, step]` 範圍

## 5. 測試流程

### 推薦執行順序
```powershell
# 1. 連線驗證 (LOAD 不會啟動，最安全)
python tests\01_connection_test.py

# 2. 互動測試 — 用 cc/on/m/off 試手感
python tests\02_manual_control.py

# 3. V-I 曲線 (建議先把 current_stop 設小)
python tests\03_vi_curve.py

# 4. 效率曲線
python tests\04_efficiency_test.py

# 5. 四模式自動掃描
python tests\05_load_sweep.py
```

### 02_manual_control.py 指令速查
| 指令 | 用途 |
|---|---|
| `idn` | 讀 *IDN? |
| `cc 5.0` | CC 模式設 5A |
| `cr 10.0` | CR 模式設 10Ω |
| `cv 24.0` | CV 模式設 24V |
| `cp 1000` | CP 模式設 1000W |
| `on` | LOAD ON |
| `off` | LOAD OFF |
| `m` | 讀 V/I/P |
| `raw <SCPI>` | 直接送原始指令（含 `?` 會自動讀回應） |
| `q` | 結束 |

## 7. 疑難排解

### Q1. 連線失敗 — `[Errno 10060] 連接嘗試失敗`
- `ping 192.168.0.100` 是否通？
- PEL-5000C 前面板 LAN 燈是否亮？
- 防火牆是否擋了 Port 4001？（暫時關掉測試）

### Q2. 連到了但 `*IDN?` 沒回應
- 檢查是否進入 `REMOTE` 模式（前面板會顯示 RMT 字樣）
- 框架在 `connect()` 後會自動送 `REMOTE`，若手動操作過儀器需先重新連線

### Q3. LOAD ON 後電流值跟設定差很多
- 確認接線無鬆動、線徑足夠
- 確認模式正確（CC 設定值是電流，CV 是電壓）
- DUT 內阻太高時，CC 模式可能拉不到指定電流，這時電流會被卡在 DUT 能提供的最大值

### Q4. 圖表中文字變方塊
- 確認 PC 有「微軟正黑體」（Windows 內建）
- 或修改 `utils/plotter.py` 的 `font.sans-serif` 列表

### Q5. 程式中斷後 LOAD 沒關
- 立刻在儀器前面板按 `LOAD` 鍵手動關閉
- 然後重新跑 `01_connection_test.py` 即可恢復遠端控制

## 8. SCPI 進階參考

### 動態模式 (Dynamic Mode)
適合測脈衝負載的場景（例如 EV 啟動瞬間）：
```python
load.write("DYN:HIGH 20")     # 高電流 20A
load.write("DYN:LOW 2")       # 低電流 2A
load.write("DYN:T1 5e-3")     # T1 5ms
load.write("DYN:T2 5e-3")     # T2 5ms
load.dynamic(True)
```

### 短路測試
```python
load.short(True)   # 開啟短路
# ... 短時間 ...
load.short(False)
```

### Sequence / Program 模式
PEL-5000C 支援儀器內建程式編輯（最多 100 步），可從面板設好後用：
```python
load.write("PROG:RUN")
```
本框架目前未封裝，可用 `raw` 直接下指令。
