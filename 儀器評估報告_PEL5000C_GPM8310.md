## 一、PEL-5000C 系列 高功率電子負載

### 1.1 設備概述
PEL-5000C 是用於 DC 電源供應器與電池的測試、評估及老化測試的高功率電子負載。
本機為「會吸電」的負載端設備，主要任務是模擬負載條件，並可同時量測輸入端的電壓、電流、功率。

### 1.2 主要操作模式
- **CC (Constant Current)**：定電流模式，不論輸入電壓如何，吸收固定電流
- **CR (Constant Resistance)**：定電阻模式，依輸入電壓線性吸收電流
- **CV (Constant Voltage)**：定電壓模式，將輸入電壓拉到設定值
- **CP (Constant Power)**：定功率模式，吸收固定功率（V × I）
- **Dynamic 動態模式**：可在 CC/CR/CP 下做脈衝負載模擬
- **Short 短路測試**

### 1.3 量測能力
| 量測項目 | 範圍 | 解析度 | 準確度 |
|---------|------|--------|--------|
| 電壓 | 0~150V / 0~600V / 0~1200V (依機型) | 5 位數 | ± 0.025% of (Reading + Range) |
| 電流 | 5mA ~ 2000A (依機型) | 5 位數 | ± 0.05% of (Reading + Range) |
| 功率 | 6kW ~ 24kW (依機型) | 5 位數 | ± 0.06% of (Reading + Range) |

### 1.4 關鍵特色
- 16-bit 電壓/電流量測
- 獨立可調的電流上升/下降斜率 (Slew Rate)
- 內建脈衝產生器，可做動態負載
- CC+CV / CP+CV 複合模式
- 類比編程輸入 (0~10V Analog Programming Input)
- 電流監控輸出 (I-Monitor BNC)
- 遠端電壓感測 (V-sense)：補償長負載線壓降
- Master/Slave 並聯：1 主 7 從，總共最多 8 台並聯擴充功率

### 1.5 保護機制
- OVP 過電壓保護（額定 105%）
- OCP 過電流保護（額定 104%）
- OPP 過功率保護（額定 105%）
- OTP 過溫保護（約 90°C）
- 反向極性保護

### 1.6 進階測試功能
- **OCP 測試**：階梯式電流爬升驗證電源過電流保護點
- **OPP 測試**：階梯式功率爬升驗證電源過功率保護點
- **Short 測試**：短路電流量測
- **Battery Discharge**：電池放電容量量測（CC/CP），可記錄 AH/WH
- **MPPT 測試**：太陽能最大功率點追蹤
- **Surge 測試**：突波電流測試
- **Auto Sequence**：可編輯 9 組程式 × 16 步驟的自動序列測試
- 150 組記憶體儲存/呼叫設定

### 1.7 通訊介面
- 標準：USB / RS-232 / LAN / GPIB（皆為選配卡）
- 後面板 BNC：Analog Input / I-Monitor / V-Sense

### 1.8 機型功率範圍
| 系列 | 功率範圍 | 電壓選項 | 電流上限 |
|------|---------|---------|---------|
| PEL-5006C ~ PEL-5024C | 6kW ~ 24kW | 150V / 600V / 1200V | 最高 2000A |

## 二、GPM-8310 數位功率計

### 2.1 設備概述
GPM-8310 是高精度、可編程的數位功率計，專門用於量測低功率裝置（交換式電源、變壓器、電源供應器、適配器等）的待機功耗。
特別適用於符合 IEC 62301 待機功耗測試標準與 EUP Lot6 法規。

### 2.2 量測規格
| 項目 | 規格 |
|------|------|
| 電壓範圍 | 15V / 30V / 60V / 150V / 300V / 600V (6 檔) |
| 電流範圍 | 5mA ~ 20A (12 檔) |
| 外部感測輸入 | EXT1 (10V max) / EXT2 (2V max) |
| 準確度 | 0.1% of reading + 0.05% of range |
| 頻寬 | DC ~ 100kHz |
| A/D 解析度 | 16-bit，取樣率約 300kHz |
| 顯示 | 5 位數 |

### 2.3 主要量測參數
- 基本電氣量：電壓 (V)、電流 (I)、功率 (P, VA, VAR)
- 功率特性：功率因數 (PF)、相位角 (DEG)、頻率 (VHz/IHz)
- 峰值：V±pk、I±pk、P±pk
- 失真分析：總諧波失真 THDV / THDI（最高 50 階）
- 波峰因數 (Crest Factor)：可設 3 / 6 / 6A
- 數學運算 (MATH)：A+B、A-B、A×B、A÷B、A÷B²、A²÷B

### 2.4 顯示模式
- 標準模式：2 主要 + 8 次要量測值同時顯示
- 簡易模式：4 個主要量測值放大顯示
- 波形顯示：V/I/P 波形圖（最高 10kHz）
- 諧波分析：Bar 圖 + List 表格（最多 50 階）

### 2.5 積分 (Integrator) 功能
- 三種模式：Manual（手動）、Standard（標準）、Continuous（連續循環）
- 積分項目：瓦時 Wh（WP/WP+/WP-）、安時 Ah（q/q+/q-）
- 計時範圍：1 秒 ~ 9999 小時

### 2.6 通訊介面
- 標準：USB / RS-232 / LAN / GPIB
- 選配：Digital I/O / DA4（4 通道 D/A 輸出 ±5V）

## 三、兩台儀器功能比較

### 3.1 量測準確度比較（以 DC 量測為基準）

| 項目 | PEL-5000C | GPM-8310 | 較佳者 |
|------|-----------|----------|--------|
| 電壓準確度 | ± 0.025% + Range | ± 0.1% + 0.05% Range | **PEL-5000C** |
| 電流準確度 | ± 0.05% + Range | ± 0.1% + 0.05% Range | **PEL-5000C** |
| 功率準確度 | ± 0.06% + Range | ± 0.1% + 0.05% Range | **PEL-5000C** |
| 量測位數 | 5 位數 | 5 位數 | 相同 |

> 重點：PEL-5000C 在純 DC 量測上反而比 GPM-8310 更準確。

### 3.2 功能差異比較

| 項目 | PEL-5000C | GPM-8310 |
|------|-----------|----------|
| 主要角色 | 「吸電」+ 順便量測 | 專門量測 |
| 頻寬 | 主要為 DC + 工頻 | DC ~ 100kHz |
| 諧波分析 | 無 | 最高 50 階 |
| THD 失真分析 | 無 | THDV / THDI |
| 真有效值 (True RMS) | 基本量測 | 含波峰因數 3/6/6A |
| 功率因數 PF | 不直接顯示 | 有 |
| 相位角 | 無 | 有 |
| 波形顯示 | 無 | 最高 10kHz |
| 積分計算 (Wh/Ah) | 有（電池放電累積容量） | 完整 Integrator 模式 |
| 波形失真量測 | 無 | 有 |
| 可作為負載使用 | 是（主要功能） | 否（僅量測） |

### 3.3 定位差異總結

- **PEL-5000C**：量測是附屬於負載控制的回饋功能，主要看「我正在吸多少電」
- **GPM-8310**：專業功率分析儀，量測本身就是核心，可分析非線性負載、開關電源、整流波形

## 四、DC 發電機應用分析

### 4.1 為何現代主流是 AC 發電機而仍有 DC 發電應用？

#### 主流為 AC 發電機的原因
- 遠距離傳輸：AC 可用變壓器升降壓，傳輸損耗低
- 結構簡單：AC 同步發電機免電刷/換向器，免維護
- 大功率容易：可做到 GW 等級
- 頻率同步：整個電網可同步運轉

#### 現代「DC 發電」的實際架構
多數 DC 發電系統實際上是 **AC 發電 + 整流器** 架構：
```
[轉子旋轉] → [產生 AC] → [整流橋] → [DC 輸出]
```

### 4.2 DC 發電機/系統的常見應用

| 應用 | 功率範圍 | 說明 |
|------|---------|------|
| 車輛交流發電機 (Alternator) | 數百 W ~ 2kW | 引擎驅動，三相 AC + 整流，輸出 12V/24V DC |
| 小型風力發電機 | 100W ~ 數 kW | AC 發電 + 整流 + 充電池 |
| 太陽能板 | 不限 | 本質為 DC 電源 |
| 微型水力發電機 | 小功率 | 永磁 AC 發電 + 整流 |
| 手搖/腳踏發電機 | 數 W ~ 數百 W | 教學/應急 |
| 工業用直流馬達電源 | 大功率 | 傳統用途，多被 VFD 取代 |
| 鐵路機車（柴電） | 高功率 | 柴油 → AC → 整流 → DC 牽引馬達 |
| 航空電源系統 | 中小功率 | 28V DC 系統 |
| 焊接發電機 | 中功率 | 引擎驅動，工地用 |
| 電動車馬達回充 | 高功率 | 馬達當發電機，量回充效率 |

### 4.3 DC 發電機核心測試項目

| 測試項目 | 內容 | PEL-5000C 可做？ |
|---------|------|-----------------|
| 負載特性曲線 (V-I curve) | 不同負載電流下的輸出電壓 | 是 |
| 功率輸出曲線 (P-I 或 P-V) | 找最大功率輸出點 | 是 |
| 效率曲線 (η vs Load) | 不同負載下的轉換效率 | 是（電端） |
| 轉速 vs 輸出 | RPM 對 V/I/P 的響應 | 需配合外部轉速計 |
| 暫態響應 | 負載突變的恢復時間 | 是 |
| 輸出漣波 | 整流品質 | 需配合示波器 |

## 五、DC 系統與功率因數的關係

### 5.1 結論：純 DC 系統沒有功率因數問題
功率因數 (PF) 是 AC 系統獨有的概念，DC 系統 PF 永遠 = 1。

### 5.2 為什麼 DC 沒有 PF 問題？

| PF < 1 的成因 | 為何 DC 不會發生 |
|-------------|----------------|
| 相位差 (cos φ) | DC 沒有頻率，沒有相位的概念 |
| 波形失真 (諧波) | DC 是平直波形，沒有諧波 |

### 5.3 但 DC 發電機有「類似」概念要注意

#### 輸出漣波 (Ripple)
- DC 發電機（特別是有刷直流發電機、整流後的交流發電機）輸出不是純 DC
- 漣波會造成電流 RMS 值 > 平均值
- 概念類似 PF，稱為「漣波因數 (Ripple Factor)」或「Form Factor」

```
理想 DC：────────────  (純直流)
實際 DC：∿∿∿∿∿∿∿∿∿  (DC + AC 漣波)
```

#### 何時需要量漣波？
- 電池充電器：漣波會影響電池壽命
- 精密電子設備供電：漣波會干擾類比電路
- 整流後的 DC：本質上會有 6 倍頻漣波
- 車用 Alternator：交流發電 + 整流，有明顯漣波

#### 量漣波應使用什麼工具？
- 第一選擇：示波器（最直觀）
- GPM-8310 的諧波分析功能也可參考，但非首選

## 六、採購建議結論

### 6.1 針對 DC 發電機發電功效量測

| 評估項目 | 建議 |
|---------|------|
| GPM-8310 是否需要購買？ | **不需要** |
| 主要原因 | DC 量測上 PEL-5000C 的準確度反而較佳；GPM-8310 強項（諧波/PF/相位/波形）對 DC 沒有意義 |
| PEL-5000C 是否足夠？ | **足夠**，內建量測功能可滿足需求 |

### 6.2 GPM-8310 強項在 DC 應用上是否能發揮？

| GPM-8310 賣點 | DC 應用上的價值 |
|-------------|----------------|
| 100kHz 頻寬 | 不適用，DC 無頻率成分 |
| 諧波分析（50 階） | 不適用，DC 沒有諧波 |
| THD 失真分析 | 不適用，DC 沒有失真概念 |
| 功率因數 PF | 不適用，DC 永遠是 1 |
| 相位角分析 | 不適用，DC 無相位 |
| 真有效值 (True RMS) | 不適用，DC 直接讀值即可 |
| 波形顯示 | 不適用，平直一條線沒意義 |

### 6.3 推薦的 DC 發電機測試架構

```
[原動機/馬達] ──機械軸功率──→ [DC 發電機] ──電功率──→ [PEL-5000C 模擬負載]
   ↑                                            ↓
轉矩計+轉速計                              直接量 V/I/P
(這才是真正需要                            計算電氣輸出功率
 額外加購的設備)
```

**發電效率公式**：
$$\eta = \frac{\text{電氣輸出功率}}{\text{機械輸入功率}} \times 100\%$$

### 6.4 預算分配建議
若預算有剩，建議改投資以下項目（比再買 GPM-8310 更實用）：
- 高精度轉矩感測器 + 轉速計（量機械輸入功率）
- 高精度 DC 電流分流器 (Shunt) + 高位數 DMM（追求 ppm 級準確度時）
- 資料擷取系統 (DAQ)，記錄長時間效率曲線
- 示波器（觀察輸出漣波）

### 6.5 何時才需要追加 GPM-8310？
僅在以下狀況才考慮：
1. 發電機輸出有顯著漣波，需頻譜分析
2. 要做正式法規認證報告，需溯源至專業功率計
3. 暫態響應分析（負載突變時的瞬間波形紀錄）
4. 將來會擴展到 AC 應用（變頻器、馬達控制器等）

## 七、PEL-5000C 電腦整合方案

### 7.1 通訊介面選項

| 介面 | 速度 | 距離 | 多台整合 | 推薦度 | 備註 |
|------|------|------|---------|--------|------|
| GPIB | 快 | ≤20m | 最多 14 台 | 中 | 工業傳統，需 GPIB 卡 |
| RS-232 | 9.6k~115.2k bps | 短 | 1 對 1 | 低 | 老設備整合用 |
| USB | USB 2.0 | ≤5m | 1 對 1 | 高 | 最方便，免驅動 |
| LAN | 100Mbps | 不限 | 多台共網 | **最高** | 最推薦 |

### 7.2 最推薦方案：LAN (Ethernet)

#### 優勢
- 不限距離，可遠端監控
- 支援多台儀器同時連接同一網路
- 不需驅動程式
- 可直接用 TCP Socket 通訊

#### 設定步驟
1. 接 RJ-45 網路線到後面板 LAN port
2. 用內附 IPScanner.EXE 掃描設備並設定 IP
3. 設定 Socket Port 為 4001 (TCP Server)
4. 鮑率設 115200

### 7.3 次推薦方案：USB

#### 優勢
- 隨插即用
- 不需網路設定
- 適合單台桌面測試

#### 運作方式
- USB 在 PEL-5000C 內部會虛擬成 RS-232 COM Port（CDC class）
- 安裝 PL-2303 驅動後，電腦會看到一個虛擬 COM 埠
- 用 RS-232 一樣的方式控制

## 八、SCPI 與 Python 整合說明

### 8.1 核心觀念：SCPI 只是「文字指令」

SCPI (Standard Commands for Programmable Instruments) 本質上只是一串純文字字串，它不挑語言。

```
MEAS:VOLT?
```

這就是一個 SCPI 指令，只有 11 個 ASCII 字元。任何能傳送/接收文字的程式語言都能用 SCPI。

### 8.2 完整通訊鏈

```
┌──────────────┐    ┌──────────┐    ┌──────────┐    ┌────────────┐
│ Python 程式  │ → │ 通訊套件 │ → │ 作業系統 │ → │  PEL-5000C │
│              │    │          │    │  驅動程式│    │   儀器     │
│ "MEAS:VOLT?" │    │ pyserial │    │  COM 埠  │    │            │
└──────────────┘    └──────────┘    └──────────┘    └────────────┘
```

### 8.3 Python 對應套件

| 介面 | Python 套件 | 用途 |
|------|------------|------|
| RS-232 / USB-CDC | `pyserial` | 串列通訊 |
| LAN (TCP) | `socket` (內建) | 網路通訊 |
| GPIB / 通用 | `pyvisa` | VISA 標準 |

### 8.4 範例程式碼

#### 範例 A：USB / RS-232（pyserial）
```python
import serial

ser = serial.Serial('COM3', 115200, timeout=1)
ser.write(b'MEAS:VOLT?\n')
result = ser.readline().decode()
print(result)  # "12.345"
ser.close()
```

#### 範例 B：LAN（socket）
```python
import socket

s = socket.socket()
s.connect(('192.168.1.100', 4001))
s.sendall(b'MEAS:VOLT?\n')
result = s.recv(1024).decode()
print(result)  # "12.345"
s.close()
```

#### 範例 C：GPIB（pyvisa）
```python
import pyvisa

rm = pyvisa.ResourceManager()
inst = rm.open_resource('GPIB0::5::INSTR')
result = inst.query('MEAS:VOLT?')
print(result)  # "12.345"
```

### 8.5 PEL-5000C 常用 SCPI 指令速查

| 指令 | 功能 |
|------|------|
| `REMOTE` | 進入遠端控制模式 |
| `LOCAL` | 離開遠端模式 |
| `MODE CC` | 設定為 CC 模式 |
| `CC:HIGH 20` | 設定電流 20A |
| `CR:HIGH 100` | 設定電阻 100Ω |
| `CV:HIGH 24` | 設定電壓 24V |
| `CP:HIGH 500` | 設定功率 500W |
| `LOAD ON` | 開啟負載 |
| `LOAD OFF` | 關閉負載 |
| `MEAS:VOLT?` | 讀取電壓 |
| `MEAS:CURR?` | 讀取電流 |
| `MEAS:POW?` | 讀取功率 |
| `MEAS:VC?` | 同時讀電壓電流 |
| `*IDN?` | 查詢儀器型號 |

### 8.6 DC 發電機 V-I 曲線自動測試完整範例

```python
import socket
import time
import csv
from datetime import datetime

class PEL5000C:
    def __init__(self, ip, port=4001):
        self.s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
        self.s.connect((ip, port))
        self.send('REMOTE')

    def send(self, cmd):
        self.s.sendall((cmd + '\n').encode())
        time.sleep(0.1)

    def query(self, cmd):
        self.send(cmd)
        return self.s.recv(1024).decode().strip()

    def set_cc(self, current_A):
        self.send('MODE CC')
        self.send(f'CC:HIGH {current_A}')

    def load_on(self):
        self.send('LOAD ON')

    def load_off(self):
        self.send('LOAD OFF')

    def measure(self):
        v = float(self.query('MEAS:VOLT?'))
        i = float(self.query('MEAS:CURR?'))
        p = float(self.query('MEAS:POW?'))
        return v, i, p

    def close(self):
        self.send('LOCAL')
        self.s.close()


# === DC 發電機 V-I 曲線測試 ===
load = PEL5000C('192.168.1.100')

# 從 0.5A 掃到 10A，每 0.5A 一個點
test_currents = [i * 0.5 for i in range(1, 21)]

filename = f'generator_test_{datetime.now():%Y%m%d_%H%M%S}.csv'
with open(filename, 'w', newline='') as f:
    writer = csv.writer(f)
    writer.writerow(['Set_I (A)', 'Measured_V (V)', 'Measured_I (A)', 'Power (W)'])

    for i_set in test_currents:
        load.set_cc(i_set)
        load.load_on()
        time.sleep(2)        # 等待穩定
        v, i, p = load.measure()
        writer.writerow([i_set, v, i, p])
        print(f'I_set={i_set:.1f}A → V={v:.3f}V, I={i:.3f}A, P={p:.2f}W')

load.load_off()
load.close()
print('測試完成，資料已存檔')
```

### 8.8 注意事項

- **指令終止符**：傳送時加 `\n` (LF)，PEL 預設可接受 LF / CR+LF
- **時序**：設定指令之間要間隔 50-100ms，避免命令塞太快
- **REMOTE/LOCAL**：RS-232/USB 用完要 `LOCAL` 解除，否則前面板會鎖定
- **錯誤處理**：可以查 `ERR?` 看是否有指令錯誤
- **量測穩定性**：負載剛切換時量測值會跳，建議等 1-2 秒再讀

### 9.1 核心結論
