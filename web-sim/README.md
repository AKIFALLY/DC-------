# DC 300A 直流充電馬達測試器 — 網頁模擬版

這是把 PyQt6 桌面程式（`ui/main_window.py`）改寫成的**純前端網頁模擬器**。
不需連接任何真實設備，內建三台儀器的模擬模型，讓使用者直接在瀏覽器中體驗完整操作流程。

> 線上展示：部署到 GitHub Pages 後即可分享連結（見下方「部署到 GitHub Pages」）。

---

## 這是什麼

| | 真實系統 | 本模擬版 |
|---|---|---|
| 介面 | PyQt6 桌面視窗 | 純 HTML/CSS/JS 網頁 |
| 儀器 | PEL-5000C / Keyence PLC / GPM-8310（真連線） | 三台儀器以 JS 物理模型模擬 |
| 紀錄輸出 | 存成 CSV + PNG 到資料夾 | CSV 由瀏覽器下載；曲線圖顯示於頁面 |
| 用途 | 現場量測 | 教學、展示、操作練習、UI 驗收 |

畫面與按鈕邏輯 1:1 對應真實程式：連線狀態機、即時監視、手動啟用閘門、
手動控制、兩種自動測試模式、前置條件檢查、手動紀錄、即時曲線，全部重現。

## 本機預覽

因為瀏覽器的 `file://` 限制，建議用一個簡單的本機伺服器開啟（任一即可）：

```powershell
# 方法 A：Python（已安裝 Python 時最簡單）
cd web-sim
python -m http.server 8000
# 然後瀏覽器開 http://localhost:8000

# 方法 B：Node
cd web-sim
npx serve .
```

直接雙擊 `index.html` 多數情況也能跑（純前端、無 fetch），但用伺服器最保險。

## 操作流程（與真實系統相同）

1. **連線三台儀器**：分別在「PLC 連線 / 負載機連線 / 功率計連線」按「連線」。
2. **手動體驗**：
   - 按左側「**手動啟用**」開啟手動控制閘門（框會變淺綠）。
   - 「伺服 ON」→ 在「速度設定」輸入 RPM →「速度設定」送出 → 轉速表開始上升。
   - 設定電流 →「變更(套用)」→「LOAD ON」→ 觀察電壓/電流/功率/溫度即時變化。
3. **自動測試**：
   - 在「設定」選測試模式、填三階段時間/電流（或轉速）。
   - 確認右側提醒列變綠「✓ 條件齊全」（需：負載機+PLC 連線、伺服 ON、速度自動）。
   - 按「開始測試」→ 觀察階段推進、即時曲線、計時；結束自動下載 CSV。
4. **手動紀錄**：按「手動紀錄」開始被動記錄，「停止紀錄」下載 CSV。

### 🧪 模擬器控制台（頁面最下方）
真實系統中由 **PLC / 實體開關** 決定的訊號，在這裡手動切換以便體驗：
- **速度控制來源 (DM7002)**：自動 / 手動。自動測試需「自動控制」。
- **注入異常 (DM7004)**：緊急停止 / 馬達驅動器異常，可用畫面上「異常 RESET」清除。
- **環境溫度**：調整模擬起始溫度。

> 這一區只存在於模擬版，真實程式沒有。

## 部署到 GitHub Pages

1. 把整個專案（或至少 `web-sim/` 資料夾）推到 GitHub repository。
2. repo → **Settings → Pages**。
3. **Source** 選 `Deploy from a branch`，Branch 選 `main`、資料夾選 `/ (root)`，存檔。
4. 等一兩分鐘，網址會是：
   `https://<你的帳號>.github.io/<repo名稱>/web-sim/`

若希望網址不要有 `/web-sim/` 後綴，可改成把 `web-sim/` 內的檔案放到 repo 根目錄，
或在 Pages 設定選 `/docs` 並把這些檔案放進 `docs/`。

## 與真實系統的差異（刻意簡化處）

- 儀器以**物理近似模型**回應，非真實量測值（發電機 Voc ∝ 轉速、V=Voc−I·R、P=V·I、溫度隨功率升降）。
- 心跳（DM130）、PLC 上位鏈路封包、SCPI 指令等通訊細節不模擬，只重現對操作者可見的行為。
- 紀錄只輸出 CSV（瀏覽器下載）；曲線圖在頁面上呈現，未另存 PNG（可自行截圖）。
- 設定值存於瀏覽器 `localStorage`（對應桌面版的 Windows 登錄持久化）。

操作手冊請見 `docs/UI操作手冊.md`（內容對兩版皆適用）。
