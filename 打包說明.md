# DC 發電機量測系統 — 打包說明

## 產出物一覽

| 檔案 | 用途 |
|---|---|
| `dist/DCGenTester.exe` | 主程式（單一 EXE，免安裝可直接執行） |
| `dist/config.yaml` | 設定檔，須與 EXE 放在同一資料夾 |
| `installer/DCGenTester_Setup_1.0.0.exe` | Inno Setup 安裝精靈（含設定檔，適合交付給客戶） |

---

## 前置需求

```powershell
# 安裝 Python 套件（只需做一次）
python -m pip install -r requirements.txt
python -m pip install pyinstaller
```

Inno Setup 安裝包需要額外安裝 **Inno Setup 6**（免費）：
- 官網：https://jrsoftware.org/isinfo.php
- 安裝後 `ISCC.exe` 會在 `C:\Program Files (x86)\Inno Setup 6\`

---

## 修改程式後的重新打包步驟

### 方法一：一鍵打包（只產出 EXE，不含安裝精靈）

雙擊執行 `build_exe.bat`，或在終端機：

```bat
build_exe.bat
```

完成後會產出：
- `dist\DCGenTester.exe`
- `dist\config.yaml`（從根目錄複製過去）

### 方法二：手動打包（完整流程）

**步驟 1 — PyInstaller 打包主程式**

```powershell
cd "C:\Users\USER\Documents\捷耀\DC發電機量測系統"
python -m PyInstaller DCGenTester.spec --noconfirm
copy /Y config.yaml dist\config.yaml
```

> `--noconfirm` 讓打包時自動覆蓋舊的 dist/，不會問確認。

**步驟 2 — Inno Setup 打包安裝精靈**（選用，要交付給客戶才需要）

```powershell
# 方式 A：命令列（確認 ISCC.exe 路徑）
"C:\Program Files (x86)\Inno Setup 6\ISCC.exe" installer.iss

# 方式 B：用 Inno Setup Compiler GUI 開啟 installer.iss 再按 Build
```

完成後產出：`installer\DCGenTester_Setup_1.0.0.exe`

## 打包設定說明

### DCGenTester.spec（PyInstaller 設定）

| 項目 | 值 | 說明 |
|---|---|---|
| 入口點 | `ui/main_window.py` | 程式主檔 |
| 模式 | `onefile` | 所有程式碼打進單一 EXE |
| 主控台 | `console=False` | 視窗程式，不顯示黑色終端機視窗 |
| 收入 bundle | `ui/assets/logo.png` | 公司 LOGO（解壓至 `assets/logo.png`） |
| 排除套件 | pandas, openpyxl, tkinter, pytest | 主程式未用，排除以縮小體積 |

> ⚠ `config.yaml` **不**收進 EXE，刻意放在旁邊，讓現場可直接用記事本修改 IP、額定值等設定，不需要重新打包。

### installer.iss（Inno Setup 設定）

| 項目 | 值 |
|---|---|
| 版本 | 1.0.0（修改版號請同步更新此檔第 4 行 `MyAppVersion`） |
| 預設安裝位置 | `%LocalAppData%\Programs\DCGenTester`（不需管理員權限） |
| config.yaml | `onlyifdoesntexist`：重裝/升級不會蓋掉現場改過的設定 |
| 建立桌面捷徑 | 安裝時可選，預設打勾 |

---

## 更新版本號

若要發布新版本，修改 `installer.iss` 第 4 行：

```ini
#define MyAppVersion "1.0.1"
```

然後重新跑完整兩步驟打包，安裝包檔名會自動更新成 `DCGenTester_Setup_1.0.1.exe`。

## 常見問題

**Q：EXE 執行時跳「找不到 config.yaml」**
- `config.yaml` 必須和 `DCGenTester.exe` 放在**同一個資料夾**。
- 使用安裝精靈安裝的話，安裝程式會自動處理。

**Q：打包後 LOGO 不見了**
- 確認 `ui/assets/logo.png` 存在再重新打包。
- spec 裡的 `datas=[('ui/assets/logo.png', 'assets')]` 負責把它收進去。

**Q：新增/修改了 driver 或 utils 模組，打包後找不到**
- 直接重新執行 `build_exe.bat` 即可，PyInstaller 會自動掃描 import。
- 若用了新的第三方套件，先 `pip install 套件名`，再打包。
